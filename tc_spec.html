01. 초기 상태 — 페이지 로드 시 기본 렌더링 검증
TC-001 기능 페이지 로드 시, 신청 버튼 비활성화 확인 페이지 로드 완료
  1. 신청하기 버튼이 흐리게 표시되는지 확인
  2. 버튼에 표시된 텍스트 확인
 "30자 더 입력 시 신청 가능" 문구와 함께 버튼이 흐리게 비활성화되어 있음 PASS
TC-002 기능 페이지 로드 시, 글자 수 카운터 0 확인 페이지 로드 완료
  1. 입력창 위에 표시된 글자 수 확인
 글자 수 카운터가 "0 / 100"으로 표시됨 PASS
TC-003 기능 페이지 로드 시, 신청 탭이 기본 활성화 페이지 로드 완료
  1. 신청 탭이 선택된 상태인지 확인
  2. 취소 패널이 화면에 보이지 않는지 확인
 신청 탭이 선택(활성화)된 상태이고, 취소 패널은 화면에 보이지 않음 PASS
TC-004 기능 페이지 로드 시, 폼·쿠폰 메시지 비노출 페이지 로드 완료
  1. 신청 폼 아래 메시지가 보이지 않는지 확인
  2. 쿠폰 영역 안내 메시지가 보이지 않는지 확인
 신청 메시지와 쿠폰 메시지 모두 화면에 표시되지 않음 PASS
02. 입력값 유효성 — 경계값 분석(BVA) · 동등분할
TC-005 기능 29자 입력 시 버튼 비활성화 (MIN-1) 페이지 로드 완료
  1. 입력창에 29자를 입력
  2. 신청하기 버튼이 비활성화 상태인지 확인
  3. 버튼에 표시된 텍스트 확인
 신청하기 버튼이 비활성화되고 "1자 더 입력 시 신청 가능"으로 표시됨 PASS
TC-006 기능 30자 입력 시 버튼 활성화 (MIN 경계) 페이지 로드 완료
  1. 입력창에 30자를 입력
  2. 신청하기 버튼이 활성화되는지 확인
  3. 버튼에 표시된 텍스트 확인
 신청하기 버튼이 활성화되고 텍스트가 "체험단 신청하기"로 변경됨 PASS
TC-007 기능 31자 입력 시 버튼 활성화 (MIN+1) 페이지 로드 완료
  1. 입력창에 31자를 입력
  2. 신청하기 버튼이 활성화되는지 확인
 신청하기 버튼이 활성화됨 PASS
TC-008 기능 100자 입력 시 버튼 활성화 (MAX 경계) 페이지 로드 완료
  1. 입력창에 100자를 입력
  2. 신청하기 버튼이 활성화되는지 확인
  3. 글자 수 카운터가 "100 / 100"인지 확인
 신청하기 버튼이 활성화되고 카운터가 "100 / 100"으로 표시됨 PASS
TC-009 기능 공백만 입력 시 버튼 비활성화 페이지 로드 완료
  1. 입력창에 공백 60자를 입력
  2. 신청하기 버튼이 비활성화 상태인지 확인
  3. 글자 수 카운터가 "0 / 100"인지 확인
 공백은 글자로 인식되지 않아 버튼이 비활성화 상태이고 카운터는 "0 / 100" 유지 PASS
TC-010 기능 앞 공백 포함 유효 30자 → 버튼 활성화 페이지 로드 완료
  1. 입력창에 공백 3자 + 한글 30자를 입력
  2. 신청하기 버튼이 활성화되는지 확인
 앞뒤 공백을 제외하고 30자로 인식해 신청 버튼이 활성화됨 PASS
TC-011 기능 maxlength=100 초과 입력 차단 페이지 로드 완료
  1. 입력창에 101자를 붙여넣기
  2. 실제로 입력된 글자 수 확인
 100자까지만 입력되고 101번째 문자는 자동으로 차단됨 PASS
03. 체험단 신청 흐름
TC-012 기능 유효 입력 후 신청 → 성공 메시지 노출 입력창에 30자 이상 입력 완료
  1. 신청하기 버튼 클릭
  2. 신청 폼 아래 메시지 내용 확인
 성공 메시지 "체험단 신청이 정상적으로 접수되었습니다."가 표시됨 PASS
TC-013 기능 신청 성공 후 신청 인원 1 증가 입력창에 30자 이상 입력 완료
  1. 신청하기 버튼 클릭
  2. 화면에 표시된 신청 인원 수 확인
 신청 인원이 0명 → 1명으로 증가함 PASS
TC-014 기능 신청 성공 후 버튼 비활성화 + 텍스트 변경 입력창에 30자 이상 입력 완료
  1. 신청하기 버튼 클릭
  2. 버튼이 비활성화됐는지 확인
  3. 버튼에 표시된 텍스트 확인
 버튼이 비활성화되고 텍스트가 "신청 완료"로 변경됨 PASS
TC-015 기능 이미 신청 완료 상태에서 재신청 시 오류 메시지 신청 완료 상태
  1. 테스트를 위해 버튼을 임시로 활성화
  2. 신청 버튼 재클릭
  3. 오류 메시지 내용 확인
 오류 메시지 "이미 신청이 완료된 상태입니다."가 표시됨 PASS
TC-016 기능 29자 입력 상태에서 신청 버튼 클릭 불가 페이지 로드 완료
  1. 입력창에 29자를 입력
  2. 신청 버튼 클릭 시도
  3. 버튼이 비활성화 상태인지 확인
 버튼이 비활성화 상태로 클릭 자체가 되지 않음 PASS
04. 탭 전환 — 신청 탭 ↔ 취소 탭
TC-017 기능 취소 탭 클릭 → 취소 패널 노출·신청 패널 숨김 페이지 로드 완료, 신청 탭 활성화 상태
  1. 취소 탭 클릭
  2. 취소 패널이 화면에 나타나는지 확인
  3. 신청 패널이 화면에서 사라지는지 확인
 취소 패널이 화면에 표시되고, 신청 패널은 숨겨짐 PASS
TC-018 기능 신청 전 취소 탭 → "내역 없음" 안내 메시지 신청하지 않은 상태
  1. 취소 탭 클릭
  2. 취소 탭 안내 메시지 내용 확인
 안내 메시지 "아직 신청된 내역이 없습니다."가 표시됨 PASS
TC-019 기능 신청 전 취소 탭 → 취소 버튼 비활성화 신청하지 않은 상태
  1. 취소 탭 클릭
  2. 취소 버튼이 비활성화 상태인지 확인
 신청 내역이 없으므로 취소 버튼이 비활성화 상태로 표시됨 PASS
TC-020 기능 신청 후 취소 탭 → 제출 내용 노출 신청 완료 상태
  1. 취소 탭 클릭
  2. 기존에 제출한 신청 내용이 표시되는지 확인
 신청 시 입력한 텍스트가 그대로 표시됨 PASS
TC-021 기능 탭 전환 후 신청 완료 상태 유지 신청 완료 상태
  1. 취소 탭 클릭
  2. 신청 탭 다시 클릭
  3. 신청하기 버튼 상태와 텍스트 확인
 탭을 전환해도 버튼은 여전히 비활성화 상태이며 "신청 완료" 텍스트 유지됨 PASS
05. 신청 취소 흐름
TC-022 기능 취소 성공 → 성공 메시지 노출 신청 완료 상태, 취소 탭 활성화
  1. 취소 버튼 클릭
  2. 취소 완료 메시지 내용 확인
 성공 메시지 "신청이 취소되었습니다."가 표시됨 PASS
TC-023 기능 취소 후 신청 인원 1 감소 신청 완료 상태 (신청 인원 1명), 취소 탭 활성화
  1. 취소 버튼 클릭
  2. 화면에 표시된 신청 인원 수 확인
 신청 인원이 1명 → 0명으로 감소함 PASS
TC-024 기능 취소 후 신청 탭 폼 전체 초기화 취소 처리 완료 상태
  1. 신청 탭 클릭
  2. 입력창이 비어있는지 확인
  3. 글자 수 카운터 확인
  4. 신청하기 버튼 상태 확인
 입력창이 초기화되고, 카운터가 0이며, 신청 버튼이 비활성화 상태임 PASS
TC-025 기능 취소 후 재신청 가능 취소 처리 완료, 폼 초기화 상태
  1. 신청 탭 클릭
  2. 입력창에 30자를 입력
  3. 신청 버튼 클릭
  4. 신청 인원 수 확인
 재신청이 정상 처리되고 신청 인원이 다시 1명으로 증가함 PASS
TC-026 기능 취소 후 카운터 음수 방지 (0 미만 불가) 취소 완료 상태 (신청 인원 0명)
  1. 테스트를 위해 취소 버튼을 임시로 활성화
  2. 취소 버튼 재클릭
  3. 신청 인원 수 확인
 신청 인원이 0명 아래로 내려가지 않고 0명을 유지함 PASS
06. 쿠폰 발급 — 상태 전이 테스트
TC-027 기능 쿠폰 받기 클릭 → 발급 안내 메시지 노출 쿠폰을 아직 받지 않은 상태
  1. 쿠폰 받기 버튼 클릭
  2. 쿠폰 영역 안내 메시지 내용 확인
 안내 메시지 "10% 할인 쿠폰이 발급되었습니다."가 표시됨 PASS
TC-028 기능 발급 후 버튼 비활성화 + 텍스트 "발급 완료" 쿠폰을 아직 받지 않은 상태
  1. 쿠폰 받기 버튼 클릭
  2. 버튼이 비활성화됐는지 확인
  3. 버튼에 표시된 텍스트 확인
 버튼이 비활성화되고 텍스트가 "발급 완료"로 변경됨 PASS
TC-029 기능 중복 쿠폰 발급 시 오류 메시지 이미 쿠폰을 받은 상태
  1. 테스트를 위해 버튼을 임시로 활성화
  2. 쿠폰 받기 버튼 재클릭
  3. 오류 메시지 내용 확인
 오류 메시지 "이미 발급된 쿠폰입니다."가 표시됨 PASS
TC-030 기능 쿠폰 발급 상태와 신청 상태 독립 동작 이미 쿠폰을 받은 상태
  1. 입력창에 30자를 입력
  2. 신청하기 버튼이 활성화되는지 확인
 쿠폰 수령 여부와 관계없이 신청 버튼이 정상적으로 활성화됨 PASS
07. 관련 상품 링크
TC-031 기능 관련 상품 클릭 → 외부 링크 새 탭 열림 페이지 로드 완료, 인터넷 연결 정상
  1. 관련 상품 카드 클릭
  2. 새 탭이 열리는지 확인
  3. 스킨푸드 상품 페이지가 정상적으로 열리는지 확인
 새 탭에서 스킨푸드 상품 페이지가 정상적으로 열림 PASS
TC-032 기능 관련 상품 이미지 로딩 실패 시 alt 텍스트 표시 이미지 파일 경로를 존재하지 않는 경로로 임시 변경
  1. 페이지 새로고침
  2. 이미지 영역 확인
  3. 이미지 대신 대체 텍스트가 표시되는지 확인
 이미지가 로드되지 않을 경우 "레몬그라스 나이아신아마이드5% 패드" 대체 텍스트가 표시됨 PASS
08. 비기능 — 반응형 UI · 접근성 · 렌더링

  1. 브라우저 너비를 1280px로 설정
  2. 페이지 접속
  3. 왼쪽·오른쪽 패널이 나란히 배치되는지 확인
 왼쪽 패널(70%)과 오른쪽 패널(30%)이 좌우로 나란히 배치됨 PASS
TC-034 비기능 태블릿(1024px) 단일 컬럼 전환 1280px 너비에서 페이지 로드 완료
  1. 브라우저 너비를 1024px로 축소
  2. 레이아웃이 세로로 변경되는지 확인
 좌·우 패널이 위아래로 쌓이는 단일 컬럼 레이아웃으로 전환됨 PASS
TC-035 비기능 모바일(375px) 카드 패딩·폰트 크기 확인 브라우저 준비 완료
  1. 브라우저 너비를 375px로 설정
  2. 카드 안쪽 여백이 적절한지 확인
  3. 글자 크기가 읽기에 적합한지 확인
 카드 여백이 20px로 유지되고 레이아웃이 깨지지 않음 PASS
TC-036 비기능 모바일에서 버튼·쿠폰박스 세로 배치 확인 브라우저 너비 375px 설정 완료
  1. 쿠폰 영역이 세로로 배치되는지 확인
  2. 버튼이 화면 전체 너비로 표시되는지 확인
 쿠폰 영역이 세로로 배치되고, 버튼이 전체 너비로 표시됨 PASS
TC-037 비기능 모든 이미지에 alt 속성 존재 (접근성) 페이지 소스 접근 가능
  1. 페이지 내 모든 이미지 태그 확인
  2. 각 이미지에 대체 텍스트가 있는지 확인
 상품 이미지 4개(Lpad5_User.png, Howto_Use.png, Lpad5.jpg, Confirm.jpg) 모두 대체 텍스트 존재 PASS
TC-038 비기능 textarea label for·id 매핑 (접근성) 페이지 로드 완료
  1. 입력창과 레이블이 올바르게 연결되어 있는지 확인
  2. 레이블 텍스트("신청 이유를 입력해주세요") 클릭
